# 微博内容归档工具

> 一键保存任意微博用户的内容为本地 Markdown 文件，支持多用户批量操作。
> 提供图形界面和命令行两种方式，适合个人数据备份和本地阅读。

---

## 能做什么

| 功能 | 说明 |
|------|------|
| **多用户批量保存** | 同时保存多位博主的内容，各自独立归档 |
| **智能增量更新** | 再次运行只获取新发布的内容，已保存的不再重复 |
| **原创/转发筛选** | 可选择仅保存原创内容，或同时保存转发内容 |
| **图片自动下载** | 图片以 Base64 内嵌到文档中，单文件自包含 |
| **按周自动分拆** | 内容按 ISO 周分目录存放，便于回溯查阅 |
| **断点续传** | 中途可暂停/继续/停止，已获取的数据不会丢失 |
| **快速测试** | 先获取 3 条验证设置是否正确 |
| **关键词过滤** | 多关键词（逗号/顿号/空格/分号分隔），不区分大小写，正文/话题含任一即保存 |
| **关键词分组管理** | 保存为命名分组（如"蔚来相关"）、一键应用、从文件批量导入、最近使用记录 |
| **用户分组管理** | 保存为命名分组（如"蔚来黑子"）、一键应用/替换/合并、从文件批量导入、最近使用记录 |
| **已抓取用户列表** | 自动生成统计报告，含每个用户基本信息、粉丝数、抓取微博数 |
| **大文件分拆** | 超过 10MB 的文档可自动按标题分割为小文件 |

## 图形界面说明

启动后看到主窗口，分为四个区域：

### 📋 目标用户区
- **添加**：输入微博用户 ID 和昵称，添加到待处理列表
- **移除**：从列表中删除选中的用户
- 用户列表显示 **ID / 昵称 / 已保存条数 / 最近更新时间**
- **👥 用户分组**：可将当前列表保存为命名分组，后续一键应用（替换或合并）
  - 支持从 .txt 文件批量导入用户（每行 `user_id nickname`）
  - 支持管理分组（编辑/删除）
  - 最近使用记录快速恢复

### 🔑 登录信息区
- 点击编辑按钮，粘贴浏览器中的 Cookie 字符串（从微博开发者工具获取）
- 不添加也可使用，但部分内容可能无法获取
- 点击标签查看 Cookie 有效性

### ⚙️ 参数配置区
- **起始日期**：只获取该日期之后发布的内容
- **内容模式**：全部（原创+转发）或仅原创
- **分拆方式**：按周 / 按月 / 按天
- **图片下载**：是否下载原创/转发中的图片
- **保存格式**：Markdown / JSON / CSV

### ▶ 操作控制区
| 按钮 | 说明 |
|------|------|
| 开始采集 | 按配置开始获取所有用户的内容 |
| 暂停 | 暂停当前任务（可稍后恢复） |
| 继续 | 恢复已暂停的任务 |
| 停止 | 安全停止并保存已获取的数据 |
| 测试(3条) | 仅获取选中用户的最新 3 条，快速验证 |
| ✂ 分割大文档 | 扫描超过 10MB 的 Markdown 文件，按二级标题自动分割 |

## 输出说明

所有内容保存在 `weibo_data/用户昵称/` 目录下，结构如下：

```
weibo_data/
└── Dear-迪丽热巴/
    ├── 2026-W01.md      # 2026年第1周的内容
    ├── 2026-W02.md      # 2026年第2周的内容
    └── INDEX.md         # 目录索引
```

每篇内容包含：
- 发布时间（精确到秒）
- 正文（文字+话题+@用户）
- 图片（Base64 内嵌，无需外网）
- 互动数据（点赞/转发/评论数）

## 辅助功能

### 📝 手动链接保存
在「手动链接」文本框粘贴微博链接（每行一条，最多 20 条），点击「批量保存为 MD」，即可逐条获取并保存。

支持格式：
- `https://weibo.com/用户ID/微博ID`
- `https://m.weibo.cn/detail/微博ID`

### ✂ 大文件分拆
当单周/单月内容超过 10MB 时，可点击「✂ 分割大文档」按钮，自动按二级标题（## ）边界切割为多个小文件，方便阅读。

## 命令行模式

适合服务器或定时任务场景：

```bash
# 按配置文件运行
python weibo.py

# 配置文件示例 config.json
{
    "user_id_list": ["1669879400"],
    "only_crawl_original": 1,
    "since_date": "2024-01-01",
    "write_mode": ["markdown"],
    "original_pic_download": 1,
    "cookie": "你的Cookie（可选）"
}
```

## 注意事项

1. **数据仅供个人备份使用**，请尊重原作者版权，勿用于商业用途
2. **访问频率保护**：工具内置随机延迟，避免对平台造成压力
3. **Cookie 有效期**：如长时间未使用，可能需要重新获取 Cookie
4. **网络环境**：需要能够正常访问微博网站
